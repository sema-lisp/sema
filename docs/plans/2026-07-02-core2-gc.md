# CORE-2: Cycle-reclaiming memory strategy — design + de-risking spike plan

**Status:** design accepted (ADR #66), spike not started
**Branch:** `worktree-core2-gc-design`
**Scope:** design + oracle tests + spike plan. No collector code lands from this document; the spike plan below is the first implementation step.

---

## 1. Problem, measured

Sema's runtime is single-threaded, reference-counted (`Rc`, never `Arc`), with NaN-boxed
`Value(u64)` whose ~25 heap variants are all `Rc<T>` (`crates/sema-core/src/value.rs:612`,
`685`). Reclamation is pure `Rc` drop (`impl Drop for Value`, value.rs:1786 — the single
choke point where every heap tag's `Rc` is released). There is no tracing, no finalization,
no cycle collection. Reference cycles are therefore immortal.

Three concrete cycle shapes exist **today** (all confirmed by measurement on this branch;
oracle tests in `crates/sema/tests/leak_test.rs`):

### Shape U — VM upvalue self-capture (the churn leak)

A recursive **local** closure captures its own name as an `Rc<UpvalueCell>` whose
`UpvalueState::Closed(Value)` holds the closure (`crates/sema-vm/src/vm.rs:31-53`; lowering
at `crates/sema-vm/src/resolve.rs:280-302` deliberately pre-registers the local slot so the
body can self-reference).

```
Rc<NativeFn> ──payload──▶ VmClosurePayload ──▶ Rc<Closure>
      ▲                                             │ upvalues[i]
      │                                             ▼
      └────────── Closed(Value) ◀────────── Rc<UpvalueCell>
```

**Measured:** `(define (churn) (define (loop n) …(loop …)) (loop 3))` in a loop leaks
**260 B/iteration** in-process (counting allocator), 25 MB → 314 MB RSS over 1M iterations
(debug build; control workload without self-capture stays flat at 25 MB). This is the shape
that bites long-running agents: every agent turn that defines a recursive local helper leaks
it forever.

### Shape E — Env ⇄ Closure via home globals (the teardown leak)

Every `MakeClosure` concretizes `Closure::globals = Some(home_env)` (vm.rs:2971-2983 — this
is the M1 "home globals" design that makes exported module fns resolve their private
helpers). `DEFINE_GLOBAL` (vm.rs:1656) then binds the closure into that same env:

```
Rc<Env> ─bindings─▶ SpurMap ─▶ Value(NativeFn) ─payload─▶ Closure ─globals─▶ same Rc<Env>
```

So **every top-level `(define (f …) …)` creates an env⇄closure cycle**. Harmless while the
env is the process-lifetime global env; fatal the moment an env is meant to die: notebook
kernel reset (`crates/sema-notebook/src/engine.rs:340`), embedded `Interpreter` teardown,
module-cache eviction (future). **Measured:** one user define pins the ENTIRE global env —
every builtin `NativeFn`, the bindings map, everything: **~168 KB leaked per
`Interpreter` create→define→drop cycle**.

This shape is the VM successor of the task's "tree-walker `Lambda { env }`" variant. The
original variant is **dead code on live paths today**: `Value::lambda` is constructed only
in `#[cfg(test)]` code (`sema-llm/src/builtins.rs:7681+`, `sema-vm/src/serialize.rs:1683`),
and `call_value` rejects raw Lambdas (`sema-eval/src/eval.rs:376`). The design below still
covers `Lambda` in the trace model (it's cheap), but the real target is Env ⇄ Closure.

### Shape D — builtin delegates strongly capture their home env (found during this design)

`register_vm_delegates` (`crates/sema-eval/src/eval.rs:553`) and the deftool/defagent
registrars install ~11 `NativeFn`s (`__vm-eval`, `__vm-load`, `__vm-import`,
`__vm-defmacro`, `__vm-define-multi`, `__vm-define-record-type`, force/macroexpand/tool/
agent delegates — `let …_env = env.clone()` at eval.rs:558, 611, 633, 658, 704, 720, 764,
793, 923, 945) whose **boxed `Fn` strongly captures the very `Rc<Env>` they are registered
into**:

```
Rc<Env> ─bindings─▶ Value(NativeFn "__vm-eval") ─Box<dyn Fn> capture─▶ same Rc<Env>
```

**Measured: an `Interpreter` that evaluates NOTHING leaks ~166 KB on drop.** This shape is
worse than U/E for a collector because the capture hides inside an opaque `Box<dyn Fn>` —
no collector can trace it. It must be fixed by refactor (§6 M1), not by collection, and it
sets a convention the collector's correctness depends on (§5 invariant I2).

### Where it bites

CLI/script runs never notice (process exit reclaims). Long-lived sessions do: REPL,
notebook server, HTTP server, MCP server, and — the language's thesis — long-running agents
that create recursive local closures every turn. At the measured 260 B/iter, an agent
creating ~40 recursive helpers/second leaks ~1 MB/minute, unboundedly.

## 2. Why cycles are structurally narrow in Sema (the mutable-edge invariant)

Safe Rust cannot construct a cyclic `Rc` graph out of immutable constructors — a cycle
needs a back-edge written **after** the target exists. Auditing every heap type
(full table in §3), post-construction `Value` mutation exists in exactly **six** places:

| Mutable cell | Location | Written by |
|---|---|---|
| `Env.bindings: Rc<RefCell<SpurMap<Spur, Value>>>` | value.rs:2367 | `define`/`set!`/module load |
| `UpvalueCell.state: RefCell<UpvalueState>` | vm.rs:41 | upvalue close, `set!` through capture |
| `Thunk.forced: RefCell<Option<Value>>` | value.rs:198 | `force` memoization |
| `AsyncPromise.state: RefCell<PromiseState>` | value.rs:237 | promise resolution |
| `Channel.buffer: RefCell<VecDeque<Value>>` | value.rs:263 | `channel/send` |
| `MultiMethod.{methods,default}: RefCell<…>` | value.rs:364 | `defmethod` |

Plus one construction-time special case: `MakeClosure` (vm.rs:2911) creates the self-edge —
but it, too, routes through an `UpvalueCell` (the closure's own-name cell), so it is covered
by the cell above. And one **opaque** case: `Box<dyn Fn>` captures inside `NativeFn.func`
(shape D) — eliminated by refactor + convention, see invariant I2.

**Invariant I1 (load-bearing):** every reference cycle in a Sema heap passes through at
least one of the six severable cells above. Therefore a collector never needs to free
memory or mutate immutable structures: it reclaims cycles by **severing** — clearing the
cell (`Closed(v)` → `Closed(NIL)`, `bindings.clear()`, …) — and letting ordinary `Rc` drop
cascade from there. No `unsafe`, no double-free risk, no change to `Value`'s representation.

**Invariant I2 (must be established by M1 and then held):** a `NativeFn`'s boxed `func`
must not strongly capture anything that can transitively hold a `Value` or `Env`. Anything
traceable a native fn needs must live in `NativeFn.payload` (traced via a registered
payload tracer) or be captured weakly. Today the only violators are the ~11 delegates
(shape D) and `make_closure`'s fallback captures (which duplicate the payload — see §5.3).

## 3. Trace / reachability model

Node identity = the `Rc` allocation's data pointer. "Trace" = enumerate outgoing edges to
other heap nodes with **exact multiplicity** (trial deletion counts references; undercount
⇒ leak stays [safe], overcount ⇒ frees live data [disaster]).

### Cycle-capable types (must be traced; severable cells in bold)

| Type | Children (edges) | Severed how |
|---|---|---|
| `Env` (via `Rc<Env>` and the shared `bindings` Rc) | **each `Value` in `bindings`**; `parent: Option<Rc<Env>>` | `bindings.borrow_mut().clear()` + `bump_version()` |
| `UpvalueCell` | **`Closed(Value)`: 1 edge** (Open: none — points into a VM stack, which is a root and shows up as external count) | `*state.borrow_mut() = Closed(NIL)` |
| `Closure` (sema-vm) | `func: Rc<Function>`; each `upvalues[i]: Rc<UpvalueCell>`; `globals: Option<Rc<Env>>`; `functions` table (1 edge; the table holds 1 edge per `Function`, and each `Function.chunk.consts: Vec<Value>` is real edges — usually reader literals, but macro expansion compiles raw expansion values, closures included, into consts via lower_expr's catch-all `CoreExpr::Const`) | not directly; its cells/env are |
| `NativeFn` with `is_closure` (VM closure wrapper) | `payload → VmClosurePayload → Closure` ×1; fallback `Box<dyn Fn>` captures `Rc<Closure>` ×1, `Rc<Env>`(home) ×1, functions ×1 (vm.rs:2988-2990) — **multiplicities fixed by the M2 capture-simplification**, see §5.3 | not directly |
| `Thunk` | `body: Value`; **`forced: RefCell<Option<Value>>`** | clear `forced` (and `body`→NIL is not possible — body is immutable; severing `forced` suffices: a cycle through a thunk must re-enter via `forced`, since `body` is set pre-construction) |
| `AsyncPromise` | **`Resolved(Value)`** | `*state.borrow_mut() = Resolved(NIL)` — an unreachable promise has no observers by definition, so the state change is unobservable |
| `Channel` | **each buffered `Value`** | `buffer.borrow_mut().clear()` |
| `MultiMethod` | `dispatch_fn: Value`; **`methods` map keys+values; `default`** | clear `methods`/`default` |
| `Lambda` (legacy, test-only) | `body: Vec<Value>`; `env: Env` | via its env |
| `Macro` | `body: Vec<Value>` (templates — literal data) | n/a (cannot cycle) |

### Interior (pass-through) types — traced, never severed, never registered

`List`/`Vector` (`Rc<Vec<Value>>`), `Map` (`Rc<BTreeMap<Value,Value>>`), `HashMap`,
`Record.fields`, `ToolDefinition.{parameters,handler}`, `Agent.tools` — all hold `Value`s
but are **immutable after construction**, so they can only be interior path segments of a
cycle, never the back-edge. Trace = iterate elements (via `ValueViewRef`, no refcount
churn).

### Leaf types — skipped entirely

`String`, `Bytevector`, `F64Array`, `I64Array`, big-int, `Prompt`, `Message`,
`Conversation` (String/metadata only — verified value.rs:322-343), `StreamBox` (OS handles,
no Values — verified against all `SemaStream` impls in `sema-stdlib/src/stream.rs`),
`NativeFn` without payload (builtins — safe **only** under invariant I2).

### Roots (never collected; external refs to them come out in the strong-count math)

The trial-deletion algorithm needs **no root enumeration** — a Rust-stack or VM-stack
reference is simply an external strong count the internal-edge subtraction can't account
for, so the object is kept. This is the decisive property for Sema: stdlib native fns hold
`Value`s on the Rust stack across arbitrary code, and no shadow-stack discipline is
feasible across 16 crates. The following are additionally treated as **pinned** (descent
skipped, a pure optimization — §5.2): the interpreter's root env, `Scheduler.globals`, the
DAP debugger's held values, `EvalContext.module_cache` export values.

### `Rc<dyn Any>` payloads

Workspace-wide audit: exactly **one** payload type exists — `VmClosurePayload`
(vm.rs:2984 is the sole `with_payload` producer). The collector exposes
`register_payload_tracer(TypeId, fn(&Rc<dyn Any>, &mut dyn FnMut(GcEdge)))`; sema-vm
registers the VmClosurePayload tracer at startup (same pattern as
`set_eval_callback` — sema-core stays independent of sema-vm, matching the workspace
dependency flow). Unknown payload types are treated as **externally referenced**
(never collected — conservative, safe).

## 4. Options analysis

Worked example legend — each option is evaluated against:
- **U**: `(define (churn) (define (loop n) …(loop (- n 1))) (loop 3))` × 1M — 260 B/iter today.
- **E**: `Interpreter::new()` + `(define (f x) x)` + drop — 168 KB/drop today.

### Option A — Bacon–Rajan trial deletion, decrement-buffered (textbook / PHP model)

Buffer an object as a *possible cycle root* whenever its refcount is decremented but stays
> 0; when the buffer fills (PHP: 10k), run the synchronous Bacon–Rajan collection
(MarkGray → Scan → CollectWhite) over the buffered candidates.

*Worked U:* when `churn` returns, the frame's ref to the loop-closure `NativeFn` drops;
count stays ≥1 (the cell's ref) → buffered. At threshold, MarkGray from the NativeFn
trial-decrements along NativeFn→Closure→cell→NativeFn; every count reaches 0 → white →
sever the cell → cascade frees. ✔

*Worked E:* on `Interpreter` drop, the root env's `Rc<Env>` drop leaves count ≥1 (each
closure's `globals`) — but **the env drop is an `Rc<Env>` drop, not a `Value` drop**, and
our only decrement hook is `impl Drop for Value` (value.rs:1786). The env cycle's Value
members (the closure NativeFns) are never dropped at teardown (their refs live in the
bindings map that is itself part of the cycle) → **nothing gets buffered → E is missed**.
Fixable by also hooking `Rc<Env>` drops (custom `EnvHandle` wrapper), at the cost of
touching every `Env` clone/drop site. ✘ (needs augmentation)

*Cost model:* space — buffer entry (8-16 B) per buffered object + dedup bit problem: with
no header bits available (NaN-boxing leaves none, and we don't own `Rc`'s layout), dedup
needs a side hash set probe **on every buffered drop**. Time — the killer is drop
frequency: stdlib HOFs clone+drop the callback closure per element, so `(map f xs)` over
1M elements buffers 1M times (hash-probe each) and triggers ~100 threshold collections in
the hot loop. Collection itself is O(candidate subgraph edges). Pause — bounded by
subgraph size, fine. Blast radius — `Value::drop` (hottest path in the runtime) gains a
branch + occasional side-table probe for every heap value whose count stays >0.

**Verdict: rejected.** Correct and standard, but it taxes the exact path (`Value::drop`)
the perf constraint protects, misses shape E without extra Env-drop hooks, and the dedup
side-table probe on hot drops is a measurable regression risk on HOF-heavy code.

### Option B — same detection algorithm, creation-time registry of cycle-birth objects (recommended)

Keep Bacon–Rajan's collection phase (MarkGray/Scan/CollectWhite — the standard algorithm,
unchanged), but change **candidate discovery**: instead of buffering on decrement,
maintain a registry of the only objects that can *be born into* cycles: the owners of the
six mutable cells of §2. Concretely — VM closures (at `make_closure`), envs (registered
the first time a closure adopts one as home — one check in `make_closure`), and the four
cold constructors `delay`/`async`/`channel`/`defmulti` (a data-only cycle needs no
closure: `(define t (delay (list t)))` or a channel sent to itself cycles through
`Thunk.forced`/`Channel.buffer` alone; these constructors are rare and never in numeric
hot loops, so registering them is free in practice). Upvalue cells need no registration —
a cell cycle always contains its owning closure, which is registered. Registry entries
are `Weak`, so dead objects self-prune and drops cost **zero**.

Collection = for each live registry entry, run trial deletion on its subgraph (shared
side-map per collection pass, so overlapping subgraphs are traced once).

*Worked U:* the loop-closure is registered at `MakeClosure`. After `churn` returns, the
only strong refs are internal (cell⇄closure). At the next trigger (registry-growth
threshold or safe point), MarkGray from it zeroes the whole 3-node subgraph → white →
sever the cell → freed; the `Weak` registry entry prunes itself. Registry stays O(live
closures), not O(iterations). ✔

*Worked E:* the root env was registered when the first closure adopted it. While the
`Interpreter` lives, trial deletion finds external counts (Interpreter's own
`Rc<Env>`, scheduler, EvalContext) → stays. After `Interpreter::drop` (which triggers a
collection — an explicit safe point), the env's counts are fully internal
(closure `globals` edges + delegate captures once M1 removes the latter) → whole env
severed → 168 KB reclaimed. ✔

*Cost model:* space — one `Weak` + Vec slot (~16 B) per **live** closure/registered env;
zero headers. Time — `make_closure` gains one `Weak::new`+push (~5-10 ns, and
`make_closure` already allocates a `Closure`, `VmClosurePayload`, `NativeFn`, `Rc` ×4, so
this is noise); `Value::drop` unchanged; calls unchanged; HOF loops unchanged. Collection —
O(Σ candidate subgraph edges), with pinned-root descent-skip keeping live-closure scans to
their local capture set (~tens of ns per live closure). Pause — proportional to live
closures + garbage found; measured gate in M4, with the 300k-churn benchmark as the canary.
Blast radius — `sema-core` new module (~400 lines), `make_closure` (+3 lines), safe-point
call sites (REPL/notebook/agent/Interpreter::drop, ~5 sites), delegate refactor (M1).

**Verdict: recommended.** Same proven detection mathematics as A (it *is* Bacon–Rajan's
synchronous collector; only the root-candidate policy differs — a registry instead of a
decrement buffer, which PHP/CPython can't use because their containers mutate freely,
while Sema's cycle-birth sites are a small closed set of cold constructors), zero cost on
the drop/call hot paths, and it handles shape E naturally because envs are registered,
not discovered-by-drop.

### Option C — tracing mark-sweep over an arena / GC handles for heap types

Replace `Rc<T>` with `Gc<T>` handles into an arena for the ~25 heap types (or the
cycle-capable subset), trace from roots, sweep.

*Worked U/E:* trivially collected — that's what tracing GCs do. ✔✔

*Cost model / why it dies here:* **root enumeration.** Every stdlib native fn
(`crates/sema-stdlib/**`, hundreds of fns) holds `Value`s in Rust locals across calls back
into the evaluator; a mark phase that can't see them frees live data. The fixes are all
disqualifying: a handle-scope/shadow-stack discipline across 16 crates and every
`NativeFn` (thousands of lines of intrusive change, permanently easy to get wrong), or
conservative stack scanning (`unsafe`, platform-dependent, and NaN-boxed u64s on the Rust
stack are indistinguishable from raw pointers... they're not even pointers — payloads are
`ptr >> 3`), or pinning everything a native touches (reintroduces refcounting). Also:
`Value`'s NaN-box packs a 45-bit shifted pointer — arena handles would fit, but every
`Rc::into_raw`/`from_raw` site (value.rs:772-1826, plus serialize.rs) changes, WASM
included. Cost: months, whole-runtime risk, and the CLI hot path pays mark/sweep or
write-barrier overhead it never needed — Rc's promptness (drop-at-last-ref) is also load-
bearing for file handles/streams. **Rejected: blast radius and root-visibility are fatal
in a runtime whose FFI surface is "half the codebase".**

### Option D — scoped/region reclamation at REPL/notebook/agent-turn boundaries

Track allocations per "turn"; at the boundary, drop everything that didn't escape.

*Worked U:* churn garbage dies at the next boundary ✔ — but only if the runtime can PROVE
the loop-closure didn't escape into a global, a channel, a promise, a conversation, the
module cache… which requires exactly the reachability analysis a collector does (escape =
reachable from the persistent env), except it must be **precise about liveness, not just
cycles**: severing a non-escaped-but-still-referenced value corrupts the session.
*Worked E:* interpreter teardown is itself the boundary; "drop the world" is what `Rc`
already tries and the cycles defeat — regions don't break cycles **within** the region's
survivors. ✘
**Rejected: it's either unsound (guess escape), or it degenerates into option B's tracing
with a worse trigger policy.** The turn-boundary *idea* survives in B as safe-point
placement.

### Option E — `Weak` self-capture, revisited with a correctness story

The dropped attempt made the captured **env** weak — that broke "module exports a fn that
calls a private helper" (`vm_module_test`) because the exported fn was the only thing
keeping its home env alive; `Weak` let the env die and the helper lookup failed. Any weak
edge on the **general** ownership path has this failure mode.

The revisit: make only the **self-name upvalue cell's back-edge** weak (the cell created
for a closure's own define slot, detectable in `resolve.rs:280-302`). This survives the
module-helper case (that path is `Closure.globals`, untouched, still strong).

*Worked U:* direct self-recursion collected ✔. *But:* mutual recursion leaks —
`(define (even? n) … (odd? …))` + `(define (odd? n) … (even? …))` as locals is two cells,
neither is a *self*-capture, cycle survives ✘. `set!`-created cycles leak ✘. Data cycles
(closure in a channel it captures) leak ✘. Shape E leaks entirely ✘. And it adds a
semantic trap: a weak self-cell whose closure escaped and was dropped while the cell
survives (via `(define f …)` then `(set! f other)` then calling a stored copy... the copy
IS the closure — upgrade actually always succeeds when invoked *through the closure*;
the risk is a **stored self-cell** obtained before the rebind). Making one binding weak
by position is a bespoke semantics no other Lisp has — exactly what "follow standards"
prohibits. **Rejected as primary; not even worth keeping as an adjunct once B exists.**

### Option F — off-the-shelf crates

- **`gc` (rust-gc):** requires `Gc<T>` + `#[derive(Trace)]` on every stored type — same
  handle-replacement blast radius as option C, plus `Trace` can't be derived for
  `Rc<dyn Any>` payloads, `Box<dyn Fn>` captures, or `hashbrown` maps without newtype
  wrappers everywhere. NaN-boxing needs a stable raw-pointer round-trip (`Rc::into_raw`/
  `from_raw`); `Gc` has no equivalent contract.
- **`bacon_rajan_cc` (`Cc<T>`):** the right algorithm, wrong carrier — replaces `Rc`
  wholesale (same ~1,800-callsite migration the Spur change cost, but with `Trace` impls
  for 25 types), and its `Cc` is not `#[repr(transparent)]`-compatible with the
  `into_raw >> 3` NaN-box encoding without auditing its internals forever.
- **`broom`, `shredder`, …:** arena/concurrent designs; same root-enumeration problem as C
  (or threads, which the constraint excludes).

**Rejected.** The *algorithm* is off-the-shelf (Bacon–Rajan 2001, §3 synchronous variant —
the same one PHP ships in `ext/standard/gc` and CPython approximates); the *carrier* can't
be, because no crate collects **through** `std::rc::Rc` — and `Rc` is the one thing we
cannot swap cheaply. Hand-rolling the ~400-line collection phase against a published,
proven algorithm is the standards-following move here, and it is what the reference
implementations do.

## 5. Recommended design (option B, in full)

Two independent fixes, ordered by risk:

### 5.1 M1 — delegate capture refactor (shape D; no collector involved)

The ~11 `register_vm_delegates`/tool/agent registrars change `let x_env = env.clone()` to
`let x_env = Rc::downgrade(env)`, upgrading at call time with a clear eval error on
failure ("evaluator environment has been torn down"). This is semantically honest — a
delegate registered *into* an env must not own it — and unlike the failed Lambda-env
`Weak`, the delegate is host infrastructure: it is only callable through the env that owns
it (compiled code resolves `__vm-*` as globals in that env), so a failed upgrade is
unreachable in practice; the error path is defense, not semantics. `vm_module_test` is
untouched: module helpers resolve via `Closure.globals`, which stays strong.

This alone turns `interpreter_teardown_without_defines_is_bounded` green and establishes
invariant I2 (record it in AGENTS.md: *native fns must not strongly capture `Env`/`Value`
outside `payload`*).

### 5.2 The cycle collector (shapes U + E)

A new `sema-core/src/cycle.rs` (thread-local state, like the interner and callbacks):

- **Registry:** `Vec<GcRoot>` where `GcRoot = { kind, weak }` — `Weak<NativeFn>` for VM
  closures, `Weak<RefCell<SpurMap<Spur,Value>>>` for env bindings (env node identity is
  the *bindings* allocation: `Env` is `Clone`-by-value and multiple `Env` handles share
  one bindings Rc — value.rs:2366-2396). Registered at: `make_closure` (the new closure;
  plus its home env on first adoption — one `bool`-check via a small seen-set or an
  env-side `Cell<bool>`… decided in spike: seen-set in the registry, no core-type change),
  and the four cold data-cycle constructors (`delay`, promise creation, `channel`,
  `defmulti`) for closure-free cycles (§4 option B).
- **Tracer:** `fn trace_value(v: &Value, sink: &mut dyn FnMut(GcEdge))` matching
  `ValueViewRef` (borrowing — no refcount churn), plus `register_payload_tracer` for
  `VmClosurePayload` (registered by sema-vm at startup alongside its `__vm-*` delegates;
  same callback pattern as `set_eval_callback`, preserving the sema-core ⊀ sema-vm
  dependency direction). Exact edge multiplicities per §3's table.
- **Collection (synchronous Bacon–Rajan over candidates):** snapshot live registry entries
  → MarkGray (DFS, side `HashMap<NodePtr, {count, color}>` seeded from `Rc::strong_count`,
  decrement per internal edge) → Scan (count > 0 ⇒ black, re-increment reachable) →
  CollectWhite (sever every white node's cell per §3's "severed how" column, **including
  `bump_version()` on severed envs** so any surviving VM inline-cache entry keyed on env
  version cannot serve a stale read — vm caches are version-guarded, value.rs:2389-2396)
  → drop the side map; `Rc` cascade does the freeing.
- **Pinned descent-skip:** collection takes a pin-set of env-bindings pointers (interpreter
  root env, `Scheduler.globals`, module-cache-reachable envs); MarkGray does not descend
  into pinned nodes. Pure optimization: pinned nodes keep their external counts and are
  black by construction. Without it, tracing any live closure would walk the entire global
  namespace per collection.
- **Safe points + trigger:** `maybe_collect()` at (a) REPL top-level eval return, (b)
  notebook cell eval return, (c) agent-loop turn boundary, (d) scheduler idle (all tasks
  parked), (e) `Interpreter::drop`, (f) a `(gc)` builtin (also `(gc/stats)`), plus a
  registry-growth threshold (collect when live+dead entries exceed
  `max(1024, 2 × survivors_of_last_collect)` — CPython's generation-0 heuristic
  flattened to one generation). Invariant: safe points hold no `RefCell` borrows of any
  env/cell (assert with `try_borrow` in debug builds); collection never runs user code
  (severing drops values; no finalizers exist in Sema — and none are being added).
- **Reentrancy guard:** a `collecting: Cell<bool>`; `maybe_collect` during a collection is
  a no-op (severing cascades `Value::drop`s, which must not re-enter).
- **WASM:** same code (single-threaded, no threads used); playground sessions are
  long-lived so they benefit directly.

### 5.3 `make_closure` capture simplification (precision prerequisite)

Today the fallback `Box<dyn Fn>` captures `Rc<Closure>` + `Rc<Env>` + functions
*separately* (vm.rs:2988-2990), giving `NativeFn → Closure` multiplicity 2 and an extra
untraceable `Env` edge inside the box — a violation of I2 and a multiplicity landmine.
Spike step M2.1 changes the box to capture the `Rc<VmClosurePayload>` itself (deriving
closure/functions/globals from it at call time — `globals` is always `Some` for
MakeClosure-built closures, vm.rs:2979). Result: `NativeFn`'s edges are exactly
`payload ×2` (field + box), both traced through the one registered tracer. No behavior
change; `vm_async_test`/C1 paths exercise the fallback thoroughly.

### 5.4 What stays out (explicitly)

- **No per-object headers, no color bits in `Value`** — NaN-boxing untouched; all
  collection state is a transient side map.
- **No changes to `Rc` semantics anywhere** — promptness of non-cyclic drops (file
  handles, streams) is preserved; the collector only ever touches garbage.
- **No finalization** — severing runs no user code.
- **Generations, incrementality, concurrent collection** — YAGNI until the M4 pause
  numbers say otherwise; the algorithm has published incremental extensions if ever
  needed.

### Why this is right for Sema's thesis

Long-running agents are the product. The collector's costs land only where that product
lives (closure creation, session boundaries) and are zero on the paths where Sema is
already 100-250× off C (numeric loops allocate no closures; `Value::drop`/call dispatch
untouched). The failure mode of the prior attempt (weakening a load-bearing strong edge)
is structurally excluded: the strong graph is never altered, so `vm_module_test`'s
export-calls-private-helper pattern holds by construction — the module env survives
because the module cache and the exported closure hold real strong refs; the collector
frees it only when *nothing* outside the cycle can ever reach it again.

   ```rust
   /// Identity of a traced heap allocation: the Rc's data pointer.
   pub struct NodePtr(*const u8);

   pub enum GcNode {
       ClosureFn(Weak<NativeFn>),                        // VM closure wrapper
       EnvBindings(Weak<RefCell<SpurMap<Spur, Value>>>), // env node identity
       Thunk(Weak<Thunk>),                               // data-only cycle owners:

       Channel(Weak<Channel>),                           //   (cold) constructors
       MultiMethod(Weak<MultiMethod>),
       // upvalue cells and interior Value nodes are DISCOVERED during trace
       // (via GcEdge), never registered
   }

   /// One outgoing reference, reported once per strong Rc held (exact
   /// multiplicity — trial deletion is arithmetic on these).
   pub enum GcEdge<'a> {
       Value(&'a Value),
       Env(&'a Rc<Env>),
       /// A sema-vm-owned node (UpvalueCell) sema-core can't type: identity +
       /// current strong count + how to enumerate its children and sever it.
       Opaque {
           ptr: NodePtr,
           strong_count: usize,
           trace: fn(NodePtr, &mut dyn FnMut(GcEdge)),
           sever: fn(NodePtr),
       },
   }

   /// Registered by sema-vm at startup (same pattern as set_eval_callback).
   /// Reports each outgoing edge of a payload, with exact multiplicity.
   pub type PayloadTracer = fn(&Rc<dyn Any>, &mut dyn FnMut(GcEdge));
   pub fn register_payload_tracer(type_id: TypeId, tracer: PayloadTracer);

   /// Candidate registration — called from make_closure. Weak: never keeps
   /// garbage alive, self-prunes at collect.
   pub fn register_candidate(node: GcNode);

   #[derive(Debug, Default, Clone, Copy)]
   pub struct GcStats {
       pub candidates: usize,   // live registry entries scanned
       pub traced: usize,       // nodes visited
       pub collected: usize,    // white nodes severed
       pub pruned: usize,       // dead Weak entries removed
   }

   /// Run a full synchronous collection. `pins` = env-bindings pointers whose
   /// interiors are not descended into (session roots). Caller guarantees the
   /// safe-point invariant (no outstanding env/cell borrows; asserted in debug).
   pub fn collect(pins: &[NodePtr]) -> GcStats;

   /// Threshold-gated collect for safe points.
   pub fn maybe_collect(pins: &[NodePtr]) -> Option<GcStats>;
   ```

### M4 — the perf gate (the design's kill criterion)

Two budgets, split by what a benchmark measures (the original single ≤2% budget
mis-modeled the churn canary — see the M0 note: the pre-collector baseline leaks every
churn cycle and does no reclamation work, so churn measures the price of collection,
not bookkeeping overhead).

A benchmark's bucket is decided by **measured collector activity** (`gc/stats` after a
full run), not by its suite label: a workload that genuinely reclaims cycles is gated
by the price-of-collection ceilings of criterion 2, whichever suite it ships in.

1. **Bookkeeping tax (hard gate, ≤2% mean regression each)** — interleaved hyperfine
   A/B vs the M0 baseline binary: `closure-storm`, `upvalue-counter`,
   `higher-order-fold`, and the `numeric` suite. Numeric benchmarks must be within
   noise, **or** show a delta <1.5% with `gc/stats` proving zero collector activity
   (all-zero stats means the delta is code layout, not GC work). These allocate
   closures the collector never reclaims (acyclic garbage self-prunes as dead
   `Weak`s; live sets are re-traced but must be amortized by the growth threshold), so
   any regression is pure per-creation/registry/pass overhead and must stay under 2%.
2. **Price of collection (`recursive-closure-churn`, plus any benchmark whose
   `gc/stats` shows real reclamation)** — absolute criteria, not
   %-of-baseline: (i) ≤350 ns per reclaimed cycle, computed as
   (head − baseline wall time) / iterations — hard ceiling 1 µs; (ii) wall-time sanity
   ceiling ≤2.5× the leaking baseline; (iii) the churn leak oracle stays green (memory
   bounded mid-eval — collection stays on the `make_closure` registry-growth threshold
   path; moving it off that path requires the oracle to still pass).
3. Pause sizing: synthetic worst case (100k live closures, then `(gc)`) — measure with
   `sys/elapsed`; document the number and the threshold policy in `docs/vm-status.md`.
4. If the gate fails: levers, in measured-value order — trigger policy (survivor
   growth factor; the floor is pinned by the churn oracle's leftover-batch bound),
   per-pass cost (pointer-keyed hashing, arena'd edge lists, scratch reuse,
   prune-only fast passes), and candidate-set size (zero-upvalue closures are exempt —
   their cycles route through registered env-wrapper candidates). The algorithm has no
   per-call/per-drop component to tune away, by construction.

**Result (2026-07-02, Apple Silicon, release, order-balanced hyperfine A/B — two
8-run invocations per benchmark with command order swapped, deltas pooled):** gate
PASSED after the levers above — storm +0.91% (1037.3 → 1046.7 ms), upvalue-counter
+1.41% (481.0 → 487.8 ms), fold −0.01% (466.6 ms), churn 121.2 → 216.7 ms =
**318 ns per reclaimed cycle** at **1.79× wall** (from +136% / ~580 ns at the first
M4 attempt); all leak oracles and the 33-test `gc_stress_test` suite green
throughout. Lever attribution (churn in-collect time per full run): pointer-keyed
Fibonacci hashing 182→168 ms; edge arena + scratch reuse + quiescent-sweep skip
168→105 ms; snapshot-seeding (adjust-set removal) ~105→103 ms. Storm total
in-collect: 50 ms → 10.4 ms (growth factor 2×→4×: 24→15.4 ms; prune-only fast
passes: 15.4→10.4 ms — 494 of 497 passes never trace); per-creation registration
fused to one TLS access + a VM-side 1-entry home-adoption cache; zero-upvalue
closures exempt from candidacy.

**Formal gate (2026-07-02, Apple Silicon, release, interleaved hyperfine A/B
`-w3 -r10`, collector HEAD incl. M5 vs pre-collector baseline `9345e0b8`): PASSED.**

|---|---|---|---|---|---|

|---|---|---|

### M5 — follow-on (post-spike, before closing CORE-2)

Notebook cell/reset + agent-turn + scheduler-idle safe points; module-cache eviction hook;
severing tests for Thunk/AsyncPromise/Channel/MultiMethod shapes; `(gc/stats)` in REPL
`:mem`; docs (`website/docs/internals/` memory page; `docs/limitations.md` CORE-2 entry

**Result (2026-07-02):**
- **Cold data-cycle constructors register candidates:** `Value::{thunk,channel,
  async_promise,multimethod}` (value.rs) + the scheduler's two raw
  `Rc::new(AsyncPromise…)` spawn sites. The parked
  `data_only_channel_cycle_collected` oracle is un-ignored and green; sibling
  garbage+live stress tests added for delay-, promise- and defmulti-shaped
  closure-free cycles (`gc_stress_test.rs`). Gotcha encoded in the
  delay/multimethod tests: a global loaded in the SAME eval is pinned by the
  VM's global-load inline cache until that VM dies (bounded mid-eval
  retention), so those tests collect in a second eval.
- **Data births run the registry-growth trigger:** `register_candidate` shares
  the threshold trigger with closure births — a push that crosses the
  threshold runs an unpinned `threshold_collect` at the constructor. Without
  it, a long eval churning channels/delays/promises/multimethods with no
  closure births retained O(total births): every dead candidate's registry
  `Weak` pins its allocation's `RcBox` (~100 B/channel; measured 795 MB RSS
  and `:registry-size 8000000` after 8M `channel/new` in one eval), and no
  safe point runs until the eval returns — which a server-style loop never
  does. Acyclic churn takes the prune-only fast path; cyclic data churn
  escalates to a full trace once live candidates exceed half the threshold.
  Oracles: `acyclic_channel_churn_growth_is_bounded` (leak_test),
  `acyclic_data_churn_bounds_registry_mid_eval` /
  `cyclic_data_churn_collected_mid_eval` (gc_stress_test). Consequence: the
  growth policy now keeps registry-at-rest under the threshold, so the gated
  boundary safe points below can no longer be forced past their gate from
  Sema code — they are abort-recovery/reap-time backstops, and their tests
  pin the end-to-end bounded-registry outcome rather than "fails with the
  hook removed".
- **Safe points:** notebook cell eval + `reset()` (snapshot released before the
  old kernel drops, then a mop-up `collect` — `reset_returns_old_kernel_memory`
  proves by `Weak` probe that the old env graph is actually freed, and fails
  against the old ordering); agent-turn boundary in `run_tool_loop`
  (`gc_maybe_collect(&[])` — sema-llm can't see the VM env, pins are pure
  optimization) with the required FakeProvider test
  (`agent_turn_boundary_collects_between_tool_turns`: correctness + gc/stats
  observation of mid-turn reclamation + unchanged message correlation);
  scheduler idle in `run_scheduler_callback` after terminal-task reaping when
  the task list is empty (NOT between polls;
  `scheduler_idle_safe_point_runs_pass_after_tasks_finish`). Both boundary
  hooks are threshold-gated backstops per the data-birth-trigger consequence
  above.
- **REPL:** `,gc` command (collect + stats print; `,help` updated). Docs:
  `docs/vm-status.md` GC section (pause numbers left as a PerfFormal

- **Deliberately not done:** module-cache eviction hook (no eviction exists —

## 8. Open questions (decided in spike, not blockers)

- Env home-adoption registration: seen-set in the registry vs a `Cell<bool>` on `Env` —
  spike measures both; seen-set preferred (no core-type change).
- `AsyncPromise` severing value: `Resolved(NIL)` vs `Cancelled` — moot for observable
  behavior (the promise is unreachable), pick whichever keeps `gc/stats` honest.
- Whether `Function.chunk.consts` can ever contain a closure: **resolved in M3 — yes.**
  A macro expanding to a raw function value is compiled into chunk consts
  (lower_expr's catch-all `CoreExpr::Const`), so the tracer reports consts as real
  edges (`Function`s and function tables are interior nodes, not leaves); left
  untraced, a consts ref is a phantom external count that pins the whole env graph.
  Oracles: `interpreter_teardown_with_macro_injected_const_frees_env` (leak_test),
  `macro_injected_*` (gc_stress_test).
