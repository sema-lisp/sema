**Goal:** A public, ergonomic host surface on `Interpreter` — submit roots, drive, cancel, shut down, receive root-tagged output — plus `RuntimeCommandHandle` as the single `Send` control surface, so every host (CLI, notebook, MCP, DAP, wasm-next) drives programs one way. Unblocks P6-3.

**Architecture:** Wrap the existing, tested private primitives — `Runtime::submit_root` / `drive` / `poll_result` / `cancel_root` (`crates/sema-vm/src/runtime/state.rs:695/861/4505/3440`), `RootHandle` (state.rs:84), `ShutdownOptions`/`ShutdownReport` (state.rs:57), `DriveState` (drive.rs:40), `Interpreter::drive_vm_on_runtime` (`crates/sema-eval/src/eval.rs:308`). Cross-thread commands ride the same inbox the executor uses for completions (`block_on_inbox`, state.rs:1063) — a command enqueue wakes a parked drive loop exactly like an External completion. No new scheduling machinery.

**Tech Stack:** Rust; existing runtime module; `std::sync::mpsc`-style inbox already in `WaitRuntime`.

## Global Constraints

- Single-threaded core (`Rc`, not `Arc`) — only `RuntimeCommandHandle` may be `Send`, and it must hold no `Rc`/`Value`.
- Invariant I2 (CORE-2 GC): nothing stored in the handle or command queue may strongly hold a `Value` or `Env`.
- Errors via `SemaError::eval()`/constructors, never raw variants.
- Comments describe the code as it stands (no change-narration).
- After any commit touching `state.rs`/`vm.rs`: run `./scripts/check-unified-runtime-inventory.sh --check`; re-stamp the map if line-drift red (do not reclassify).
- Slice exit gate: `cargo nextest run --workspace && jake examples && jake smoke-bytecode && jake lint && jake docs-check` + both runtime guard scripts.

pub struct RuntimeCommandHandle { /* Weak<Inbox> or channel Sender — Send + Sync, no Rc/Value */ }

    pub fn cancel_root(&self, root: RootId) -> bool;   // false = runtime gone

### Task 3: `RootOptions` + root-tagged `OutputEvent` + public `Interpreter` surface

**Files:**
- Modify: `crates/sema-eval/src/eval.rs`, `crates/sema-vm/src/runtime/host_api.rs`
- Test: `crates/sema/tests/host_api_test.rs` (new)

**Interfaces (Produces):**
```rust
pub struct RootOptions { pub name: Option<String>, pub capture_output: bool /* default false: inherit process stdout */ }
pub enum OutputEvent { Stdout { root: RootId, text: String }, Stderr { root: RootId, text: String } }
impl Interpreter {
    pub fn submit_str(&self, src: &str, opts: RootOptions) -> Result<RootHandle, SemaError>;   // read+compile+submit, does NOT drive
    pub fn submit_value(&self, expr: Value, opts: RootOptions) -> Result<RootHandle, SemaError>;
    pub fn drive_until_settled(&self, root: &RootHandle) -> EvalResult;                        // drive_vm_on_runtime semantics
    pub fn drive_turn(&self) -> Result<DriveState, SemaError>;                                 // one bounded turn (wasm/notebook shape)
    pub fn take_output(&self) -> Vec<OutputEvent>;                                             // drained per turn when capture_output
    pub fn command_handle(&self) -> RuntimeCommandHandle;
    pub fn shutdown(&self, opts: ShutdownOptions) -> ShutdownReport;
}
```
Output capture: route the existing print callback per-root (the runtime knows the current task's root during a quantum) — only when `capture_output`; default path untouched. TDD (in `host_api_test.rs`): submit two roots with captured output printing distinct markers interleaved via `async/sleep`; assert each `OutputEvent` carries the right root. Lifecycle test: submit → drive_turn loop → poll_result → shutdown report counts. Error test: `submit_str` with a parse error surfaces `SemaError` without poisoning the runtime (a subsequent submit works).
