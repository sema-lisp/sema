# Embedded Web Runtime Distribution Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make every `sema-lang` package and native Sema distribution contain a working, offline `sema web` runtime.

**Architecture:** Treat `crates/sema/src/web/assets/` as tracked source inputs and embed them unconditionally with `include_bytes!`. A package-boundary smoke script creates and unpacks the real `.crate`, patches only unpublished workspace dependencies to the local checkout, builds that packaged source, starts its `sema web`, and verifies the served shell.

**Tech Stack:** Cargo packaging, Rust `include_bytes!`, POSIX shell, GitHub Actions, Sema integration tests.

## Global Constraints

- `sema web` must work offline after installation.
- End users must not need Jake, npm, wasm-pack, or the source repository.
- Missing runtime assets must fail compilation, never produce a reduced binary.
- The same packaged inputs must serve crates.io, cargo-dist, shell installers, and Homebrew.
- Preserve unrelated user changes in the working tree.

### Task 1: Package-boundary regression test

**Files:**
- Create: `scripts/test-packaged-sema-web.sh`
- Modify: `.github/workflows/verify.yml`

**Interfaces:**
- Consumes: `cargo package -p sema-lang` and the workspace crates in the checkout.
- Produces: a zero-argument smoke command that exits nonzero unless the `.crate` builds a working `sema web` server.

- [ ] **Step 1: Write the failing package smoke script**

Create a temporary target and extraction directory, run `cargo package -p sema-lang --allow-dirty --no-verify`, assert all twelve `src/web/assets` files exist in the archive, add local `[patch.crates-io]` entries for Sema workspace crates, build the extracted manifest, start `sema web examples/sema-web-app/app.sema --no-open`, and poll `/` until it contains the Sema Web shell marker. Preserve server stderr on failure and clean up the process and temporary directory with `trap`.

- [ ] **Step 2: Run the script to verify it fails**

Run: `./scripts/test-packaged-sema-web.sh`

Expected: FAIL because the current `.crate` omits `src/web/assets/sema_wasm_bg.wasm`.

- [ ] **Step 3: Wire the smoke test into the reusable release gate**

Add a `Smoke test packaged web runtime` step to `.github/workflows/verify.yml` after the workspace test step:

```yaml
      - name: Smoke test packaged web runtime
        run: ./scripts/test-packaged-sema-web.sh
```

This reusable workflow gates both crates.io publishing and cargo-dist releases.

### Task 2: Mandatory embedded runtime

**Files:**
- Modify: `.gitignore`
- Modify: `crates/sema/build.rs`
- Modify: `crates/sema/src/web/runtime.rs`
- Modify: `crates/sema/src/web/mod.rs`
- Add: `crates/sema/src/web/assets/**`

**Interfaces:**
- Consumes: tracked browser runtime files regenerated by `jake wasm.web-runtime`.
- Produces: unconditional `runtime::extract() -> io::Result<PathBuf>` backed by compile-time bytes.

- [ ] **Step 1: Make the generated runtime a tracked package input**

Remove `crates/sema/src/web/assets/` from `.gitignore`. Keep the existing Jake recipe as the maintainer-only regeneration command and add all twelve generated files.

- [ ] **Step 2: Remove optional build configuration**

Delete the asset existence check, `rustc-check-cfg`, and `web_runtime` cfg emission from `crates/sema/build.rs`. Remove every `#[cfg(web_runtime)]` and `#[cfg(not(web_runtime))]` branch from `runtime.rs`; always compile the embedded asset table and extractor. `include_bytes!` becomes the build-time invariant.

- [ ] **Step 3: Remove the end-user fallback**

Delete `runtime::is_available()` and the early error in `web::run`. No installed command may mention `jake wasm.web-runtime`.

- [ ] **Step 4: Run the package smoke test to verify it passes**

Run: `./scripts/test-packaged-sema-web.sh`

Expected: PASS after building the unpacked `.crate` and receiving the application shell from `sema web`.
