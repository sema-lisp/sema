Sema Web

A Lisp for the browser, powered by WebAssembly

Overview

Sema Web brings the full Sema language to the browser with reactive state, declarative UI rendering, and scoped CSS. Write expressive Lisp code that compiles to efficient DOM updates via morphdom diffing.

Reactive State

Create signals with (state val), read them with @var, and update with (put! var val). Components re-render automatically when their dependencies change.

LLM Streaming

Use (llm/chat-stream messages opts) to get a reactive signal that accumulates tokens as they arrive from the server. Components that deref the stream signal re-render progressively.

Scoped CSS

The (css {...}) function generates unique class names and injects scoped rules. Supports nested pseudo-selectors via & prefix keys.

Getting Started

Click the chat button in the bottom-right corner to try the AI assistant. Your conversation is automatically saved to localStorage and restored on reload. The widget demonstrates reactive state, computed values, LLM streaming, scoped CSS, and component composition.

Architecture

The WASM interpreter evaluates Sema source at runtime. Reactive signals (built on @preact/signals-core) drive automatic re-rendering. SIP templates compile to virtual DOM trees that are efficiently patched into the real DOM via morphdom. Event delegation on the mount target dispatches to named Sema handler functions.
